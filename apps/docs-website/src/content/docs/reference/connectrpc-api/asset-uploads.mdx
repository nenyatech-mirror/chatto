## AssetUploadService

Creates room-scoped attachment assets through chunked ConnectRPC uploads.

<a id="chatto-api-v1-AssetUploadService-CreateUpload"></a>

### CreateUpload

Starts a room-scoped attachment upload for the authenticated user.

```http
POST /api/connect/chatto.api.v1.AssetUploadService/CreateUpload
```

<a id="chatto-api-v1-CreateUploadRequest"></a>

#### Input: CreateUploadRequest

Request to start a room-scoped attachment upload.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required room ID. |
| `filename` | `string` | Original filename supplied by the client. |
| `content_type` | `string` | MIME content type. Empty values are treated as application/octet-stream. |
| `size` | `int64` | Required total file size in bytes. |
| `sha256` | `string` | Required SHA-256 digest of the complete file, lowercase hexadecimal. |
| `thread_root_event_id` | `string` | Thread root event ID when the upload is for a thread reply. |
| `also_send_to_channel` | `bool` | Whether the eventual thread reply will also be echoed to the channel. |


<a id="chatto-api-v1-CreateUploadResponse"></a>

#### Result: CreateUploadResponse

Response after creating an upload.

| Field | Type | Description |
| --- | --- | --- |
| `upload` | [`AssetUpload`](/reference/connectrpc-api/types/#chatto-api-v1-AssetUpload) | Created upload session. |

### UploadChunk

Appends one chunk at the upload's current committed offset.

```http
POST /api/connect/chatto.api.v1.AssetUploadService/UploadChunk
```

<a id="chatto-api-v1-UploadChunkRequest"></a>

#### Input: UploadChunkRequest

Request to append one chunk to an upload.

| Field | Type | Description |
| --- | --- | --- |
| `upload_id` | `string` | Required upload session ID. |
| `offset` | `int64` | Byte offset where this chunk starts. |
| `content` | `bytes` | Chunk bytes. The server enforces a per-RPC size limit. |
| `chunk_sha256` | `string` | SHA-256 digest of this chunk, lowercase hexadecimal. |
