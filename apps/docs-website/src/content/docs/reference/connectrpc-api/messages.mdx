## MessageService

Posts messages into room and thread timelines.

<a id="chatto-api-v1-MessageService-PostMessage"></a>

### PostMessage

Posts a message for the current user. The user must be a room member and
must have message.post for room messages or message.post-in-thread for
thread replies. Echoing a thread reply also requires message.echo and
message.post.

```http
POST /api/connect/chatto.api.v1.MessageService/PostMessage
```

<a id="chatto-api-v1-PostMessageRequest"></a>

#### Input: PostMessageRequest

Request to post a message in a room or thread.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required. Room where the message should be posted. |
| `body` | `string` | Message body text. Required unless attachments or attachment_asset_ids is non-empty. |
| `attachment_asset_ids` | `repeated string` | Existing room-scoped attachment asset IDs to include with the message. |
| `thread_root_event_id` | `string` | Event ID of the thread root message when posting a thread reply. |
| `in_reply_to` | `string` | Event ID this message replies to for attribution. |
| `also_send_to_channel` | `bool` | True to also echo a thread reply into the main room timeline. |
| `mention_confirmation_token` | `string` | Short-lived token returned by a prior mention confirmation challenge. |
| `link_preview` | [`LinkPreview`](/reference/connectrpc-api/types/#chatto-api-v1-LinkPreview) | Link preview selected by the client. Server response-only LinkPreview fields are ignored. |
| `attachments` | repeated [`MessageAttachmentUpload`](/reference/connectrpc-api/types/#chatto-api-v1-MessageAttachmentUpload) | Browser-uploaded attachments to include with the message. |


<a id="chatto-api-v1-PostMessageResponse"></a>

#### Result: PostMessageResponse

Result of posting a message.

| Field | Type | Description |
| --- | --- | --- |
| `result.event` | [`RoomTimelineEvent`](/reference/connectrpc-api/types/#chatto-api-v1-RoomTimelineEvent) | Renderable timeline event for the posted message. |
| `result.mention_confirmation` | [`MentionConfirmationChallenge`](/reference/connectrpc-api/types/#chatto-api-v1-MentionConfirmationChallenge) | Challenge shown when a message would notify many people. |
| `includes` | [`RoomTimelineIncludes`](/reference/connectrpc-api/types/#chatto-api-v1-RoomTimelineIncludes) | Related entities needed to render event when a message was posted. |


<a id="chatto-api-v1-MessageService-UpdateMessage"></a>

### UpdateMessage

Edits a message body. Authors can edit their own messages within the edit
window. Non-authors need message.manage and cannot change channel echo
state.

```http
POST /api/connect/chatto.api.v1.MessageService/UpdateMessage
```

<a id="chatto-api-v1-UpdateMessageRequest"></a>

#### Input: UpdateMessageRequest

Request to edit a message body.

| Field | Type | Description |
| --- | --- | --- |
| `room_id` | `string` | Required. Room containing the message. |
| `event_id` | `string` | Required. Event ID of the message to edit. |
| `body` | `string` | New message body text. |
| `also_send_to_channel` | `optional bool` | For thread replies, whether a channel echo should exist after saving. Omit to preserve the current echo state. |


<a id="chatto-api-v1-UpdateMessageResponse"></a>

#### Result: UpdateMessageResponse

Result of editing a message.

| Field | Type | Description |
| --- | --- | --- |
| `updated` | `bool` | True when the edit was accepted. |
