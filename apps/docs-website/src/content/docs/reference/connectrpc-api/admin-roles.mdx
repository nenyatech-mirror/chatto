## AdminRoleService

Provides authenticated role reads and administrative role definition changes.

<a id="chatto-admin-v1-AdminRoleService-ListRoles"></a>

### ListRoles

Lists the role catalog including permission arrays as a finite snapshot.
Requires an authenticated user. This intentionally includes permission
arrays for clients that need to show a user's assigned roles and their
effects; use chatto.api.v1.RoleService for lightweight catalog rendering.

```http
POST /api/connect/chatto.admin.v1.AdminRoleService/ListRoles
```

<a id="chatto-admin-v1-ListRolesRequest"></a>

#### Input: ListRolesRequest

Request the role catalog.


<a id="chatto-admin-v1-ListRolesResponse"></a>

#### Result: ListRolesResponse

Finite role catalog snapshot plus viewer role-management capabilities.

| Field | Type | Description |
| --- | --- | --- |
| `roles` | repeated [`AdminRole`](/reference/connectrpc-api/types/#chatto-admin-v1-AdminRole) | Roles sorted by position. |
| `viewer_can_manage_roles` | `bool` | Whether the caller may create/update/delete role definitions. |
| `viewer_can_assign_roles` | `bool` | Whether the caller may assign/revoke roles and view role rosters. |


<a id="chatto-admin-v1-AdminRoleService-GetRole"></a>

### GetRole

Gets one role plus admin detail metadata. Returns NOT_FOUND when the
role does not exist. Requires an authenticated user; the assigned-user
roster is empty unless the caller may assign roles.

```http
POST /api/connect/chatto.admin.v1.AdminRoleService/GetRole
```
