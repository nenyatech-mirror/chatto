## ViewerService

Provides authenticated user profile, preferences, and capability state.

<a id="chatto-api-v1-ViewerService-GetViewer"></a>

### GetViewer

Returns the current authenticated viewer. This RPC requires a logged-in
user; unauthenticated callers receive an UNAUTHENTICATED error.

```http
POST /api/connect/chatto.api.v1.ViewerService/GetViewer
```

<a id="chatto-api-v1-GetViewerRequest"></a>

#### Input: GetViewerRequest

Request for the authenticated viewer snapshot.


<a id="chatto-api-v1-GetViewerResponse"></a>

#### Result: GetViewerResponse

Authenticated viewer snapshot needed by application shells.

| Field | Type | Description |
| --- | --- | --- |
| `user` | [`ViewerUser`](/reference/connectrpc-api/types/#chatto-api-v1-ViewerUser) | Current authenticated user. |
| `capabilities` | [`ViewerCapabilities`](/reference/connectrpc-api/types/#chatto-api-v1-ViewerCapabilities) | Permission-derived capabilities. |
| `server_notification_preference` | [`ServerNotificationPreference`](/reference/connectrpc-api/types/#chatto-api-v1-ServerNotificationPreference) | Server-wide notification preference. |
| `room_notification_preferences` | repeated [`RoomNotificationPreference`](/reference/connectrpc-api/types/#chatto-api-v1-RoomNotificationPreference) | Notification preferences for rooms the user participates in. |

## PresenceService

Lets clients report the authenticated user's current live presence.

<a id="chatto-api-v1-PresenceService-ReportPresence"></a>

### ReportPresence

Reports the current user's live presence status. This state is transient:
clients should refresh it periodically while visible, and should stop
calling this RPC when the user chooses to appear offline.

```http
POST /api/connect/chatto.api.v1.PresenceService/ReportPresence
```

<a id="chatto-api-v1-ReportPresenceRequest"></a>

#### Input: ReportPresenceRequest

Request to report the current user's live presence status.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Live status to report for the authenticated user. Offline is rejected. |
| `user_selected` | `bool` | True when this report comes from a deliberate user selection rather than automatic idle/refresh reporting. Automatic reports do not overwrite an active manually selected Away or Do Not Disturb status from another client. |


<a id="chatto-api-v1-ReportPresenceResponse"></a>

#### Result: ReportPresenceResponse

Result of reporting live presence.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Reportable status accepted and stored by the server. |
