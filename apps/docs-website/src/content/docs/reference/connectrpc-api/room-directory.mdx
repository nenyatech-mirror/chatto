## RoomDirectoryService

Reads room directory data.

<a id="chatto-api-v1-RoomDirectoryService-ListRooms"></a>

### ListRooms

Lists rooms visible to the current user. Channel rooms are non-archived
rooms visible through membership or room.list. DM rooms are membership-only
and empty DM conversations are omitted. Results are returned as a finite
navigation snapshot.

```http
POST /api/connect/chatto.api.v1.RoomDirectoryService/ListRooms
```

<a id="chatto-api-v1-ListRoomsRequest"></a>

#### Input: ListRoomsRequest

Request for rooms visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `scope` | [`RoomDirectoryScope`](/reference/connectrpc-api/types/#chatto-api-v1-RoomDirectoryScope) | Which room kinds to include. Defaults to all. |


<a id="chatto-api-v1-ListRoomsResponse"></a>

#### Result: ListRoomsResponse

Finite snapshot of rooms visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `rooms` | repeated [`DirectoryRoom`](/reference/connectrpc-api/types/#chatto-api-v1-DirectoryRoom) | Rooms matching the requested scope. |


<a id="chatto-api-v1-RoomDirectoryService-ListRoomGroups"></a>

### ListRoomGroups

Lists ordered channel room groups and sidebar items visible to the current
user as a finite navigation snapshot. Hidden room entries are omitted.
Archived room entries are omitted unless include_archived_rooms is true.

```http
POST /api/connect/chatto.api.v1.RoomDirectoryService/ListRoomGroups
```

<a id="chatto-api-v1-ListRoomGroupsRequest"></a>

#### Input: ListRoomGroupsRequest

Request for ordered room groups visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `include_archived_rooms` | `bool` | Include archived room entries. Ordinary sidebar clients should leave this false; management surfaces can set it when they need archived rooms for direct actions such as unarchiving. |


<a id="chatto-api-v1-ListRoomGroupsResponse"></a>

#### Result: ListRoomGroupsResponse

Finite snapshot of ordered room groups visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `groups` | repeated [`RoomGroup`](/reference/connectrpc-api/types/#chatto-api-v1-RoomGroup) | Channel room groups in sidebar order. |


<a id="chatto-api-v1-RoomDirectoryService-GetRoomGroup"></a>

### GetRoomGroup

Returns one ordered room group by ID. Hidden room entries are omitted.
Archived room entries are omitted unless include_archived_rooms is true.
Returns NOT_FOUND when the room group does not exist.

```http
POST /api/connect/chatto.api.v1.RoomDirectoryService/GetRoomGroup
```

<a id="chatto-api-v1-GetRoomGroupRequest"></a>

#### Input: GetRoomGroupRequest

Request for one ordered room group visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `group_id` | `string` | Required. Room group to resolve. |
| `include_archived_rooms` | `bool` | Include archived room entries. Ordinary sidebar clients should leave this false; management surfaces can set it when they need archived rooms for direct actions such as unarchiving. |


<a id="chatto-api-v1-GetRoomGroupResponse"></a>

#### Result: GetRoomGroupResponse

One ordered room group visible to the current user.

| Field | Type | Description |
| --- | --- | --- |
| `group` | [`RoomGroup`](/reference/connectrpc-api/types/#chatto-api-v1-RoomGroup) | Resolved room group. Hidden and archived room entries are omitted. |


<a id="chatto-api-v1-RoomDirectoryService-BatchGetRoomGroups"></a>

### BatchGetRoomGroups

Returns ordered room groups for a set of stable group IDs. Hidden room
entries are omitted. Archived room entries are omitted unless
include_archived_rooms is true. Unknown groups are omitted. Results
preserve first-seen request order and repeated IDs are de-duplicated.

```http
POST /api/connect/chatto.api.v1.RoomDirectoryService/BatchGetRoomGroups
```

<a id="chatto-api-v1-BatchGetRoomGroupsRequest"></a>

#### Input: BatchGetRoomGroupsRequest

Request visible room groups by stable group ID.

| Field | Type | Description |
| --- | --- | --- |
| `group_ids` | `repeated string` | Required room group IDs. Unknown groups are omitted from the response. |
| `include_archived_rooms` | `bool` | Include archived room entries. Ordinary sidebar clients should leave this false; management surfaces can set it when they need archived rooms for direct actions such as unarchiving. |

### GetRoom

NOT_FOUND when the room does not exist and PERMISSION_DENIED when the room
exists but is hidden from the caller.

<a id="chatto-api-v1-RoomDirectoryService-BatchGetRooms"></a>

### BatchGetRooms

Returns visible rooms for a set of stable room IDs. This is a direct
hydration API, so visible archived rooms may be returned. Unknown rooms and
rooms hidden from the caller are omitted. Results preserve first-seen
request order and repeated IDs are de-duplicated.

```http
POST /api/connect/chatto.api.v1.RoomDirectoryService/BatchGetRooms
```

<a id="chatto-api-v1-BatchGetRoomsRequest"></a>

#### Input: BatchGetRoomsRequest

Request visible rooms by stable room ID.

| Field | Type | Description |
| --- | --- | --- |
| `room_ids` | `repeated string` | Required room IDs. Unknown rooms and rooms hidden from the caller are omitted from the response. Archived rooms may be returned when the caller can still refresh them directly. |


<a id="chatto-api-v1-BatchGetRoomsResponse"></a>

#### Result: BatchGetRoomsResponse

Visible room batch response for direct room hydration.

| Field | Type | Description |
| --- | --- | --- |
| `rooms` | repeated [`DirectoryRoom`](/reference/connectrpc-api/types/#chatto-api-v1-DirectoryRoom) | Resolved rooms in first-seen request order. |
