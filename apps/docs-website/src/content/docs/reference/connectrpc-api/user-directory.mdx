## UserDirectoryService

Provides authenticated public user directory lookups.

<a id="chatto-api-v1-UserDirectoryService-GetUser"></a>

### GetUser

Gets a user by stable ID or login. Returns NOT_FOUND when the user is
unknown.

```http
POST /api/connect/chatto.api.v1.UserDirectoryService/GetUser
```

<a id="chatto-api-v1-GetUserRequest"></a>

#### Input: GetUserRequest

Request one user by stable user ID or login identifier.

| Field | Type | Description |
| --- | --- | --- |
| `target.user_id` | `string` | Target stable user ID. |
| `target.login` | `string` | Target login identifier. |
| `avatar` | [`UserAvatarOptions`](/reference/connectrpc-api/types/#chatto-api-v1-UserAvatarOptions) | Optional avatar URL transform parameters. Omit for the original avatar URL. |


<a id="chatto-api-v1-GetUserResponse"></a>

#### Result: GetUserResponse

Public authenticated user profile response.

| Field | Type | Description |
| --- | --- | --- |
| `user` | [`UserProfile`](/reference/connectrpc-api/types/#chatto-api-v1-UserProfile) | Requested user. |


<a id="chatto-api-v1-UserDirectoryService-BatchGetUsers"></a>

### BatchGetUsers

Gets public user profiles for multiple users referenced by event-focused
payloads.

```http
POST /api/connect/chatto.api.v1.UserDirectoryService/BatchGetUsers
```

<a id="chatto-api-v1-BatchGetUsersRequest"></a>

#### Input: BatchGetUsersRequest

Request public user profiles for a set of stable user IDs.

| Field | Type | Description |
| --- | --- | --- |
| `user_ids` | `repeated string` | Required user IDs. Unknown IDs are omitted from the response. |
| `avatar` | [`UserAvatarOptions`](/reference/connectrpc-api/types/#chatto-api-v1-UserAvatarOptions) | Optional avatar URL transform parameters. Omit for the original avatar URL. |


<a id="chatto-api-v1-BatchGetUsersResponse"></a>

#### Result: BatchGetUsersResponse

Batch public user profile response.

| Field | Type | Description |
| --- | --- | --- |
| `users` | repeated [`UserProfile`](/reference/connectrpc-api/types/#chatto-api-v1-UserProfile) | Found users. The server preserves first-seen request order and de-duplicates repeated IDs. |
