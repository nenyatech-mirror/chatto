## MyAccountService

Self-service account, profile, avatar, display preference, presence,
custom-status, external identity, and account lifecycle commands for the
authenticated user.

<a id="chatto-api-v1-MyAccountService-UpdateProfile"></a>

### UpdateProfile

Updates the authenticated user's login and/or display name.

```http
POST /api/connect/chatto.api.v1.MyAccountService/UpdateProfile
```

<a id="chatto-api-v1-UpdateProfileRequest"></a>

#### Input: UpdateProfileRequest

Request to update the authenticated user's profile. At least one field must
be present.

| Field | Type | Description |
| --- | --- | --- |
| `display_name` | `optional string` | New display name, when changing it. Empty clears the explicit display name. The server also rejects control and confusing invisible characters. |
| `login` | `optional string` | New login identifier, when changing it. The server accepts ASCII letters, digits, period, underscore, and hyphen, starting with a letter or digit. |


<a id="chatto-api-v1-UpdateProfileResponse"></a>

#### Result: UpdateProfileResponse

Result of a profile update.

| Field | Type | Description |
| --- | --- | --- |
| `user` | [`User`](/reference/connectrpc-api/types/#chatto-api-v1-User) | Updated user profile. |

<a id="chatto-api-v1-MyAccountService-UpdatePresence"></a>

### UpdatePresence

Updates the current user's live presence status. This state is transient:
clients should refresh it periodically while visible, and should stop
calling this RPC when the user chooses to appear offline.

```http
POST /api/connect/chatto.api.v1.MyAccountService/UpdatePresence
```

<a id="chatto-api-v1-UpdatePresenceRequest"></a>

#### Input: UpdatePresenceRequest

Request to update the current user's live presence status.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Live status to store for the authenticated user. Offline is rejected. |
| `user_selected` | `bool` | True when this update comes from a deliberate user selection rather than automatic idle/refresh updates. Automatic updates do not overwrite an active manually selected Away or Do Not Disturb status from another client. |


<a id="chatto-api-v1-UpdatePresenceResponse"></a>

#### Result: UpdatePresenceResponse

Result of updating live presence.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Reportable status accepted and stored by the server. |
