## AccountService

Self-service account, profile, avatar, display preference, presence, and
custom-status commands.

<a id="chatto-api-v1-AccountService-UpdateProfile"></a>

### UpdateProfile

Updates the authenticated user's login and/or display name.

```http
POST /api/connect/chatto.api.v1.AccountService/UpdateProfile
```

<a id="chatto-api-v1-UpdateProfileRequest"></a>

#### Input: UpdateProfileRequest

Request to update the authenticated user's profile. At least one field must
be present.

| Field | Type | Description |
| --- | --- | --- |
| `display_name` | `optional string` | New display name, when changing it. |
| `login` | `optional string` | New login identifier, when changing it. |


<a id="chatto-api-v1-UpdateProfileResponse"></a>

#### Result: UpdateProfileResponse

Result of a profile update.

| Field | Type | Description |
| --- | --- | --- |
| `user` | [`User`](/reference/connectrpc-api/types/#chatto-api-v1-User) | Updated user profile. |

<a id="chatto-api-v1-AccountService-ReportPresence"></a>

### ReportPresence

Reports the current user's live presence status. This state is transient:
clients should refresh it periodically while visible, and should stop
calling this RPC when the user chooses to appear offline.

```http
POST /api/connect/chatto.api.v1.AccountService/ReportPresence
```

<a id="chatto-api-v1-ReportPresenceRequest"></a>

#### Input: ReportPresenceRequest

Request to report the current user's live presence status.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Live status to report for the authenticated user. Offline is rejected. |
| `user_selected` | `bool` | True when this report comes from a deliberate user selection rather than automatic idle/refresh reporting. Automatic reports do not overwrite an active manually selected Away or Do Not Disturb status from another client. |


<a id="chatto-api-v1-ReportPresenceResponse"></a>

#### Result: ReportPresenceResponse

Result of reporting live presence.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`PresenceStatus`](/reference/connectrpc-api/types/#chatto-api-v1-PresenceStatus) | Reportable status accepted and stored by the server. |


<a id="chatto-api-v1-AccountService-SetCustomStatus"></a>

### SetCustomStatus

Sets or replaces the current user's custom status. Emoji and text are
required, and expires_at must be omitted or in the future.

```http
POST /api/connect/chatto.api.v1.AccountService/SetCustomStatus
```

<a id="chatto-api-v1-SetCustomStatusRequest"></a>

#### Input: SetCustomStatusRequest

Request to set or replace the current user's custom status.

| Field | Type | Description |
| --- | --- | --- |
| `emoji` | `string` | Short emoji marker displayed with the status. |
| `text` | `string` | User-written status text. |
| `expires_at` | `google.protobuf.Timestamp` | Optional future time after which clients should stop showing the status. |


<a id="chatto-api-v1-SetCustomStatusResponse"></a>

#### Result: SetCustomStatusResponse

Result of setting the current user's custom status.

| Field | Type | Description |
| --- | --- | --- |
| `status` | [`CustomUserStatus`](/reference/connectrpc-api/types/#chatto-api-v1-CustomUserStatus) | Stored custom status after validation and normalization. |
