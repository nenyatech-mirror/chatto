Every `chatto.toml` setting can be overridden with an environment variable. The naming pattern is `CHATTO_{SECTION}_{KEY}` in uppercase.

## General

<EnvVar name="CHATTO_LOG_LEVEL" toml="general.log_level" default="info">
  Log level. Values: `debug`, `info`, `warn`, `error`.
</EnvVar>

## Web Server

<EnvVar name="CHATTO_WEBSERVER_URL" toml="webserver.url" required>
  Public URL of the server, used for absolute links and OAuth redirects.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_PORT" toml="webserver.port" default="4000">
  Port to listen on.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_COOKIE_SIGNING_SECRET" toml="webserver.cookie_signing_secret" required>
  256-bit hex secret for signing session cookies. Generate with `openssl rand -hex 32`.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_WEBSOCKET_COMPRESSION" toml="webserver.websocket_compression" default="true">
  Enable WebSocket compression.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_REQUEST_LOGGING" toml="webserver.request_logging" default="false">
  Log HTTP requests.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_ALLOWED_ORIGINS" toml="webserver.allowed_origins">
  Comma-separated list of origins allowed for cross-origin requests (CORS and WebSocket). The server's own origin (from `webserver.url`) and `localhost` at the listen port are always allowed. Defaults to `*` (wildcard) when not set, allowing any origin for multi-server support — cross-origin clients authenticate via Bearer tokens, not cookies. Set explicitly to restrict which origins can access the API.
</EnvVar>

### TLS

Built-in automatic TLS via Let's Encrypt. When enabled, Chatto handles certificate provisioning and renewal automatically.

<EnvVar name="CHATTO_WEBSERVER_TLS_ENABLED" toml="webserver.tls.enabled" default="false">
  Enable automatic TLS via Let's Encrypt. Requires ports 80 and 443 (or custom ports configured below).
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_TLS_DOMAIN" toml="webserver.tls.domain">
  Domain name for the TLS certificate. Required when TLS is enabled.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_TLS_EMAIL" toml="webserver.tls.email">
  Email address for Let's Encrypt notifications. Required when TLS is enabled.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_TLS_CACHE_DIR" toml="webserver.tls.cache_dir" default=".chatto/certs">
  Directory to cache TLS certificates.
</EnvVar>

<EnvVar name="CHATTO_WEBSERVER_TLS_HTTP_PORT" toml="webserver.tls.http_port" default="80">
  Port for the HTTP server (ACME challenges and HTTPS redirect). Use a higher port if running without elevated privileges.
</EnvVar>

## Core

<EnvVar name="CHATTO_CORE_SECRET_KEY" toml="core.secret_key" required>
  256-bit hex secret for HMAC-derived bearer-token and account-flow link verifiers. Generate with `openssl rand -hex 32`. Keep this stable across restores if you want sessions and pending links to survive.
</EnvVar>

## Assets

<EnvVar name="CHATTO_CORE_ASSETS_SIGNING_SECRET" toml="core.assets.signing_secret" required>
  256-bit hex secret for signing asset URLs. Generate with `openssl rand -hex 32`.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_MAX_UPLOAD_SIZE" toml="core.assets.max_upload_size" default="25 MB">
  Max upload size for images and non-video files (e.g., `25MB`, `25 MiB`). Videos use a separate limit — see `CHATTO_VIDEO_MAX_UPLOAD_SIZE`.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_STORAGE_BACKEND" toml="core.assets.storage_backend" default="nats">
  Storage backend for file attachments. Values: `nats`, `s3`.
</EnvVar>

### S3 Storage

Only used when `storage_backend` is set to `s3`. See the [S3 Storage guide](/guides/s3-storage/) for setup instructions.

<EnvVar name="CHATTO_CORE_ASSETS_S3_ENDPOINT" toml="core.assets.s3.endpoint">
  S3 endpoint URL (e.g., `s3.amazonaws.com` for AWS, or a custom endpoint for S3-compatible services).
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_BUCKET" toml="core.assets.s3.bucket">
  S3 bucket name. Created automatically on startup if it doesn't exist.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_REGION" toml="core.assets.s3.region">
  AWS region. Optional for non-AWS S3-compatible services.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_ACCESS_KEY_ID" toml="core.assets.s3.access_key_id">
  S3 access key ID.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_SECRET_ACCESS_KEY" toml="core.assets.s3.secret_access_key">
  S3 secret access key.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_USE_SSL" toml="core.assets.s3.use_ssl" default="true">
  Use HTTPS for S3 connections.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_S3_PATH_STYLE" toml="core.assets.s3.path_style" default="false">
  Use path-style URLs instead of virtual-hosted. Required for most S3-compatible services (Cloudflare R2, Wasabi, Backblaze B2, etc.).
</EnvVar>

### Image Cache

<EnvVar name="CHATTO_CORE_ASSETS_CACHE_ENABLED" toml="core.assets.cache.enabled" default="false">
  Enable caching for resized images. Reduces CPU usage for repeated thumbnail generation.
</EnvVar>

<EnvVar name="CHATTO_CORE_ASSETS_CACHE_TTL" toml="core.assets.cache.ttl" default="7d">
  Time-to-live for cached images. Supports durations like `7d`, `1w`, `168h`.
</EnvVar>

## NATS

<EnvVar name="CHATTO_NATS_REPLICAS" toml="nats.replicas" default="1">
  Number of replicas for JetStream streams, KV buckets, and object stores. Must be an odd number (`1`, `3`, or `5`) for quorum. Set to `3` or `5` when running a NATS cluster for fault tolerance.
</EnvVar>

### Embedded Server

<EnvVar name="CHATTO_NATS_EMBEDDED_ENABLED" toml="nats.embedded.enabled" default="true">
  Enable the embedded NATS server. Set to `false` when connecting to an external NATS server.
</EnvVar>

<EnvVar name="CHATTO_NATS_EMBEDDED_PORT" toml="nats.embedded.port" default="4222">
  Embedded NATS server port. Set to `0` to disable the TCP listener (in-process only).
</EnvVar>

<EnvVar name="CHATTO_NATS_EMBEDDED_BIND_ADDRESS" toml="nats.embedded.bind_address" default="127.0.0.1">
  Embedded NATS server bind address.
</EnvVar>

<EnvVar name="CHATTO_NATS_EMBEDDED_HTTP_PORT" toml="nats.embedded.http_port" default="0">
  NATS monitoring/stats HTTP port. Set to `0` to disable. Useful for debugging with the NATS dashboard.
</EnvVar>

<EnvVar name="CHATTO_NATS_EMBEDDED_DATA_DIR" toml="nats.embedded.data_dir" default="./data">
  JetStream data directory for the embedded server.
</EnvVar>

<EnvVar name="CHATTO_NATS_EMBEDDED_AUTH_TOKEN" toml="nats.embedded.auth_token">
  Auth token for the embedded NATS server. Must match `CHATTO_NATS_CLIENT_TOKEN` when using token auth.
</EnvVar>

### Client

<EnvVar name="CHATTO_NATS_CLIENT_URL" toml="nats.client.url" default="nats://localhost:4222">
  NATS server URL to connect to. Supports comma-separated URLs for cluster failover (e.g., `nats://n1:4222,nats://n2:4222`).
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_AUTH_METHOD" toml="nats.client.auth_method" default="token">
  NATS authentication method. Values: `none`, `token`, `userpass`, `credentials`, `nkey`.
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_TOKEN" toml="nats.client.token">
  NATS auth token (when using `token` auth method). Must match `CHATTO_NATS_EMBEDDED_AUTH_TOKEN` when using the embedded server.
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_USERNAME" toml="nats.client.username">
  NATS username (when using `userpass` auth method).
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_PASSWORD" toml="nats.client.password">
  NATS password (when using `userpass` auth method).
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_CREDENTIALS_FILE" toml="nats.client.credentials_file">
  Path to `.creds` file (when using `credentials` auth method).
</EnvVar>

<EnvVar name="CHATTO_NATS_CLIENT_NKEY_SEED" toml="nats.client.nkey_seed">
  NKey seed (when using `nkey` auth method).
</EnvVar>

## Authentication

<EnvVar name="CHATTO_AUTH_DIRECT_REGISTRATION" toml="auth.direct_registration" default="true">
  Enable direct (email/password) registration. When `false`, the registration page is hidden and the registration API returns `403`. Users can still sign in via configured SSO providers.
</EnvVar>

<EnvVar name="CHATTO_AUTH_TOKEN_TTL" toml="auth.token_ttl" default="90d">
  TTL for bearer auth tokens. Supports durations like `90d`, `2160h`. Tokens older than this are automatically invalidated.
</EnvVar>

### OIDC

<EnvVar name="CHATTO_AUTH_OIDC_ENABLED" toml="auth.oidc.enabled" default="false">
  Enable OIDC sign-in (e.g. via Chatto Hub).
</EnvVar>

<EnvVar name="CHATTO_AUTH_OIDC_ISSUER_URL" toml="auth.oidc.issuer_url">
  OIDC issuer URL. Used for discovery via `/.well-known/openid-configuration`.
</EnvVar>

<EnvVar name="CHATTO_AUTH_OIDC_CLIENT_ID" toml="auth.oidc.client_id">
  OIDC client ID, obtained from your OIDC provider.
</EnvVar>

<EnvVar name="CHATTO_AUTH_OIDC_CLIENT_SECRET" toml="auth.oidc.client_secret">
  OIDC client secret.
</EnvVar>

<EnvVar name="CHATTO_AUTH_OIDC_LABEL" toml="auth.oidc.label" default="Chatto Hub">
  Button label shown on the login page for the OIDC provider.
</EnvVar>

## SMTP

<EnvVar name="CHATTO_SMTP_ENABLED" toml="smtp.enabled" default="false">
  Enable SMTP email sending.
</EnvVar>

<EnvVar name="CHATTO_SMTP_HOST" toml="smtp.host">
  SMTP server hostname.
</EnvVar>

<EnvVar name="CHATTO_SMTP_PORT" toml="smtp.port">
  SMTP port. Common values: `587` (TLS), `465` (SSL), `25` (unencrypted).
</EnvVar>

<EnvVar name="CHATTO_SMTP_USERNAME" toml="smtp.username">
  SMTP username.
</EnvVar>

<EnvVar name="CHATTO_SMTP_PASSWORD" toml="smtp.password">
  SMTP password.
</EnvVar>

<EnvVar name="CHATTO_SMTP_FROM" toml="smtp.from">
  From address for outgoing emails.
</EnvVar>

## Push Notifications

<EnvVar name="CHATTO_PUSH_ENABLED" toml="push.enabled" default="false">
  Enable Web Push notifications.
</EnvVar>

<EnvVar name="CHATTO_PUSH_VAPID_PUBLIC_KEY" toml="push.vapid_public_key">
  VAPID public key for Web Push. Generate with `npx web-push generate-vapid-keys`.
</EnvVar>

<EnvVar name="CHATTO_PUSH_VAPID_PRIVATE_KEY" toml="push.vapid_private_key">
  VAPID private key for Web Push.
</EnvVar>

<EnvVar name="CHATTO_PUSH_VAPID_SUBJECT" toml="push.vapid_subject">
  VAPID subject identifier. Must be a `mailto:` or `https:` URL.
</EnvVar>

## Video Processing

<EnvVar name="CHATTO_VIDEO_ENABLED" toml="video.enabled" default="false">
  Enable server-side video processing. When enabled, uploaded videos are transcoded into multiple quality variants for adaptive playback. Requires `ffmpeg` and `ffprobe` to be available on the server.
</EnvVar>

<EnvVar name="CHATTO_VIDEO_FFMPEG_PATH" toml="video.ffmpeg_path">
  Path to the `ffmpeg` binary. If not set, Chatto looks for `ffmpeg` in `PATH`.
</EnvVar>

<EnvVar name="CHATTO_VIDEO_FFPROBE_PATH" toml="video.ffprobe_path">
  Path to the `ffprobe` binary. If not set, Chatto looks for `ffprobe` in `PATH`.
</EnvVar>

<EnvVar name="CHATTO_VIDEO_MAX_CONCURRENT" toml="video.max_concurrent" default="2">
  Maximum number of videos to process concurrently. Higher values use more CPU and memory.
</EnvVar>

<EnvVar name="CHATTO_VIDEO_MAX_UPLOAD_SIZE" toml="video.max_upload_size" default="100 MB">
  Max upload size for video files (e.g., `100MB`, `200 MiB`). Only applies when video processing is enabled. Non-video files use `CHATTO_CORE_ASSETS_MAX_UPLOAD_SIZE` instead.
</EnvVar>

<EnvVar name="CHATTO_VIDEO_TEMP_DIR" toml="video.temp_dir">
  Directory for temporary files during transcoding. Defaults to the system temp directory.
</EnvVar>

## LiveKit (Voice & Video Calls)

<EnvVar name="CHATTO_LIVEKIT_ENABLED" toml="livekit.enabled" default="false">
  Enable LiveKit voice and video call support. Requires a running LiveKit server.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_URL" toml="livekit.url">
  LiveKit server WebSocket URL. Use `ws://` for development, `wss://` for production.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_API_KEY" toml="livekit.api_key">
  LiveKit API key for authentication.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_API_SECRET" toml="livekit.api_secret">
  LiveKit API secret for signing JWT tokens. Keep this confidential.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_WEBHOOK_URL" toml="livekit.webhook_url">
  URL where LiveKit sends webhook events. Defaults to `{webserver.url}/webhooks/livekit`.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_INSTANCE_ID" toml="livekit.instance_id">
  Unique identifier for this Chatto server process, prefixed to LiveKit room names. Required when multiple Chatto replicas share the same LiveKit cluster, so that webhook events can be routed correctly. When empty, room names use the unprefixed format.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_WEBHOOK_API_KEY" toml="livekit.webhook_api_key">
  API key that LiveKit uses to sign webhook requests. When multiple Chatto replicas share a LiveKit cluster, the webhook signing key may differ from the per-replica API key. Falls back to `CHATTO_LIVEKIT_API_KEY` when not set.
</EnvVar>

<EnvVar name="CHATTO_LIVEKIT_WEBHOOK_API_SECRET" toml="livekit.webhook_api_secret">
  API secret for validating webhook signatures. Falls back to `CHATTO_LIVEKIT_API_SECRET` when not set.
</EnvVar>

## Owners

<EnvVar name="CHATTO_OWNERS_EMAILS" toml="owners.emails">
  Comma-separated list of email addresses that confer server-owner status. A user with any verified email matching the list gets full server access (including `/admin` routes). Only verified email addresses are matched.
</EnvVar>

## Limits

Server-wide resource limits. Use `-1` for unlimited (the default), `0` to disable creation entirely, or any positive integer to cap.

<EnvVar name="CHATTO_LIMITS_MAX_USERS" toml="limits.max_users" default="-1">
  Maximum number of verified users on this server. Enforced at signup: when the verified-user count is already at the limit, new signups are rejected. Note that the check is non-atomic, so a burst of concurrent signups at the boundary can briefly overshoot by one or two.
</EnvVar>
